# Security Policy

## Reporting a Vulnerability

We take the security of NestGo seriously. If you discover a security vulnerability, please do NOT open a public issue. Instead, report it through one of the following methods:

- **Email**: Reach out to the maintainer directly (if available) or via GitHub private reporting if enabled.
- **GitHub Private Reporting**: Use the "Report a vulnerability" feature in the Security tab of the repository.

We will acknowledge your report and work to resolve the issue as quickly as possible.

## Security Updates
Critical security fixes will be released as new version tags. Please stay updated with the latest releases.
